## Recommendations

### Immediate Fix for Button.js ↔ utils-window.js

**Option 1: Move `isCmdModifierPressed` to a separate utility file**
Create a new file `src/utils-keyboard.js` or `src/utils-events.js` that contains only the keyboard/event utilities without importing `Button`:

```javascript
// src/utils-keyboard.js
export function isCmdModifierPressed(event) {
  return $.client.profile().platform === 'mac' ? event.metaKey : event.ctrlKey
}
```

Then:

- `Button.js` imports from `utils-keyboard.js`
- `utils-window.js` can still import `Button`
- No circular dependency

**Option 2: Lazy import in utils-window.js**
Instead of importing `Button` at the top of `utils-window.js`, import it only where needed using dynamic imports:

```javascript
// In utils-window.js, instead of:
// import Button from './Button'

// Use dynamic import where Button is used:
export function wrapHtml(html, options = {}) {
  // ... existing code ...
  if (callbacks) {
    Object.keys(callbacks).forEach(async (className) => {
      const { default: Button } = await import('./Button')
      // ... rest of code
    })
  }
}
```

**Option 3: Dependency Injection**
Pass the `Button` class as a parameter to functions in `utils-window.js` that need it, rather than importing it.
